Description
Symptom
Following error occurs when previewing CAP service (srv) application in BAS:

<error xmlns=http://docs.oasis-open.org/odata/ns/metadata>

<code>502</code>

<message>Error during request to remote service: connect ETIMEDOUT 10.0.4.5:20003</message>

</error>

The odata connection is working correctly when the application is deployed to SAP Cloud Foundry.

Environment
SAP Business Application Studio

Cloud Application Programming Model

Reproducing the Issue
create CAP project based on backend OData service
preview the srv application using one of the following methods:
create run configuration
choose "Preview Application" from context menu
execute "cds run" command in terminal
open the preview in new window
navigate to the collection pointing to backend odata service
Cause
CAP destination configuration is applied in the package.json. The Application Preview functionality picks up the destination configuration from the .env file or any other .envX file if configured through Run Configuration. 

The BAS proxy, which is active while application preview, is listening on localhost and 8887 port. This proxy redirects the host name formatted "<destination name>.dest" through the destination with the same name.

The debug console output shows the redirected requests, which should reflect the correct "<destinationname>.dest" pattern. The logged request can be also tested in BAS terminal using curl command line tool.

For example:

curl https://demodestination.dest/opu/odata/demoservice

Please note: "cds watch" command does not use BAS proxy.

Resolution
Create destination with the proper configuration and name ("demodestination" in this example) in the same subaccount where BAS is running.  Doublecheck the destination using BAP Service Center

Create/doublecheck the following part of the package.json configuration:

"cds": {
      ...
      "backend_odata": {
        "kind": "odata-v2",
        "model": "srv/external/backend_odata",
        "credentials": {
          "destination": "demodestination"
        }
      }
    }
}
Create/doublecheck the .env file configuration:

destinations=[{"name":"demodestination","proxyHost":"http://127.0.0.1","proxyPort":"8887","url":"http://demodestination.dest"}]

Use the following command in BAS terminal to reload subaccount destinations after changing destination settings in SAP BTP Cockpit:

curl http://localhost:8887/reload
